Ожидаемый результат: MPI-версия алгоритма маршрутизации позволит эффективно передавать сообщения между любыми узлами в гиперкубе, минимизируя количество промежуточных узлов на пути.

## 2. Problem Statement
На вход поступают:

Начальное значение (целое число)

Номер конечного процесса-получателя

Задача: Передать сообщение (значение) от процесса с рангом 0 к указанному получателю через промежуточные узлы в топологии "гиперкуб", сформировав оптимальный маршрут на основе разности двоичных представлений рангов. Маршрут должен проходить через узлы, отличающиеся ровно на один бит на каждом шаге.

## 4. Parallelization Scheme
Параллельный алгоритм реализует динамическую маршрутизацию сообщений в топологии "гиперкуб" с использованием MPI. Количество процессов должно быть степенью двойки. Алгоритм использует двоичное представление рангов процессов для определения оптимального пути:

Корневой процесс (ранг 0) инициализирует передачу сообщения к целевому процессу.

На каждом шаге вычисляется разность между текущим и целевым рангами с помощью операции XOR.

Определяется первый ненулевой бит разности (номер измерения), по которому нужно переместиться.

Сообщение передается процессу, который отличается от текущего ровно по этому измерению (операция XOR с соответствующей степенью двойки).

Каждый промежуточный процесс добавляет свой ранг в маршрут и пересылает сообщение дальше.

Когда сообщение достигает целевого процесса, он отправляет финальный маршрут обратно корневому процессу.

Корневой процесс рассылает результаты всем процессам для синхронизации.

Для граничных случаев (когда получатель совпадает с отправителем или когда в системе всего один процесс) маршрутизация не выполняется. Для обеспечения времени выполнения более 0.001 секунды добавлена искусственная вычислительная нагрузка перед началом маршрутизации.
