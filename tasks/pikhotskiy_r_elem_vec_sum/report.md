## 1. Введение
Цель работы - реализовать последовательное и параллельное вычисление суммы элементов вектора, сравнить их скорость работы и определить, когда параллельная версия работает быстрее.

## 2. Постановка задачи
На вход подается кортеж (n, vector), где:
- n - размер вектора
- vector - вектор целых чисел

Задача: вычислить сумму всех элементов вектора, используя 64-битные целые числа чтобы избежать переполнения.

## 3. Последовательный алгоритм
Простой алгоритм, который проходит по всем элементам вектора и складывает их. 
Сложность: O(n) - время растет пропорционально размеру вектора.

## 4. Параллельный алгоритм (MPI)
Алгоритм использует несколько процессов для параллельного вычисления:

1. Главный процесс сообщает всем общий размер вектора
2. Данные распределяются между процессами
3. Каждый процесс вычисляет сумму своей части
4. Результаты собираются и складываются
5. Итоговая сумма отправляется всем процессам

## 7. Заключение
Параллельная версия с MPI показала низкую эффективность для этой задачи. Основные причины:
- Операция сложения слишком простая и быстрая
- Затраты на передачу данных между процессами превышают время вычислений
- Наилучшая эффективность (7.75%) достигнута при 15 млн элементов

Для получения выгоды от распараллеливания нужны либо более сложные вычисления, либо значительно большие объемы данных.
